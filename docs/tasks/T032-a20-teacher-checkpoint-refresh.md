# T032: A20 Teacher And Checkpoint Refresh

## Objective

Refresh Oracle-like teacher, trainer-input, diagnostic checkpoint, and
calibration evidence on the accepted T031 A20 source-coverage contract.

This task determines whether broader source coverage changes teacher-target
stability or checkpoint-vs-teacher diagnostics before any deeper
model-guided-search task consumes a checkpoint.

## Current Main Baseline

Current `main` has T023 scale-up, T024 teacher-to-trainer bridge, T026
checkpoint inference, and T027 calibration. Existing accepted artifacts are
smoke-scale and should not be treated as a durable input to later tasks.

T031 completed a broader coverage refresh, but its accepted distribution
diagnosis remained Act 1 only: no Act 1 Boss starts and no later-act starts
were reached. That result is useful data evidence, but it is not sufficient for
a broad A20 teacher/checkpoint refresh.

## Dependencies

- T031 is complete.
- A maintainer has accepted a later-act/Boss coverage follow-up, or this task
  has been deliberately revised into an explicitly narrow Act-1 diagnostic
  refresh before it becomes `READY`.

## Inputs And Artifacts

Inputs must come from the accepted T031 artifact contract or be regenerated by
commands reported in this PR:

- the T031 natural source pool and coverage report;
- selected Oracle-like teacher datasets from current T023/T022 workflows;
- trainer-input v6 artifacts from the T024 bridge;
- one optional diagnostic PyTorch checkpoint written under an ignored or
  external artifact path.

The PR must report artifact identities and regeneration commands. No large
teacher dataset, trainer JSONL, checkpoint, or report is committed.

## Scope

- Select source starts only by rule-defined structural metadata from the T031
  coverage contract.
- Generate current-schema teacher scale-up artifacts at named native search
  budgets.
- Generate teacher-targeted trainer-input records for one selected budget.
- Train or load at most diagnostic checkpoints under the existing `train`
  optional dependency group and named smoke or narrow-curriculum override.
- Run teacher-guidance calibration against the generated trainer/checkpoint
  pair.
- Report policy target kind, information regime, source coverage, skipped
  rows, calibration metrics, and broad-training gate status separately.

## Out Of Scope

- Controller promotion or fixed-cohort controller comparison.
- Normal-information claims from Oracle-like teacher data.
- Live-game validation.
- New model architecture work beyond what is required to run the current
  checkpoint contract.
- Treating repeated teacher rows or budget tiers as new source coverage.
- Treating the T031 Act-1-only distribution as broad A20 evidence.

## Design Constraints

- Teacher collection remains `full_simulator_state_oracle_like`.
- Required artifact inputs must be reproducible from T031 outputs and current
  commands; do not consume another branch's temporary smoke checkpoint.
- Mixed policy target kinds, source mismatches, missing trainer provenance, or
  incompatible checkpoints must fail closed.
- PyTorch remains optional under the `train` dependency group.

## Deliverables

- Exact artifact-generation commands and identities in the PR report.
- Teacher scale-up, bridge, checkpoint, and calibration summaries.
- Documentation impact notes, usually an `experiment_log.md` entry if no code
  changes are required.
- Focused tests only if code changes are needed.

## Acceptance Criteria

- Every generated teacher/trainer/checkpoint/calibration artifact has a
  reported source identity and regeneration command.
- Calibration uses a checkpoint whose trainer-input SHA-256 provenance matches
  the generated trainer artifact.
- Teacher target kind, information regime, source coverage, and skipped rows
  are reported separately.
- Broad-training readiness is reported only through the configured gate.
- No controller-strength, normal-information, or live-game claim is made.

## Required Verification

Run documentation checks and `git diff --check` for a report-only PR. If code
changes are made, run the standard local gates from `docs/tasks/README.md`.

Run WSL source verification before any simulator collection. Run T023/T024/T027
commands on the accepted T031 source contract and report exact commands,
artifact identities, and summaries.

## Legacy Reference

Use current merged T023/T024/T026/T027/T031 workflows. Do not reuse local
smoke artifacts unless the PR reports them as explicit external inputs with
schema and provenance.

## PR Report

The PR must report task ID, consumed T031 artifact identities, generated
teacher/trainer/checkpoint/calibration artifact identities, verification
commands, calibration summaries, broad-training gate outcome, known risks, and
legacy material consulted.
