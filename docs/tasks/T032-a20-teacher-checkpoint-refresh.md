# T032: A20 Narrow Teacher And Checkpoint Diagnostic Refresh

## Objective

Refresh Oracle-like teacher, trainer-input, diagnostic checkpoint, and
calibration evidence on the accepted T039 source-coverage contract.

This is a narrow diagnostic refresh, not a broad A20 training refresh. It tests
whether the recovered T037 Act 1 Boss and Act 2 source starts produce usable
teacher targets and checkpoint-vs-teacher diagnostics before any deeper
model-guided-search task consumes a checkpoint.

## Current Main Baseline

Current `main` has T023 scale-up, T024 teacher-to-trainer bridge, T026
checkpoint inference, and T027 calibration. Existing accepted artifacts are
smoke-scale and should not be treated as a durable input to later tasks.

T039 accepted `docs/a20_later_act_boss_source_coverage_contract.md` as the
durable source-coverage contract from T037. That contract recovers a narrow
Boss/later-act source supplement: 4,688 natural battle starts, including 31 Act
1 Boss starts and 3 Act 2 starts, with the T009 broad-training gate still
closed. This task consumes that narrow contract while preserving the gate
failure as evidence.

## Dependencies

- T039 is complete.
- T023, T024, T026, and T027 are complete.

## Inputs And Artifacts

Inputs must come from the T039 source-coverage contract or be regenerated by
commands reported in this PR:

- the accepted T037 no-potion natural source pool, coverage report,
  reachability report, and shard manifest named in
  `docs/a20_later_act_boss_source_coverage_contract.md`;
- all 31 accepted Act 1 Boss source starts;
- all 3 accepted Act 2 source starts;
- a deterministic Act 1 non-Boss background set of 64 source starts selected by
  rule-defined metadata from the same pool with named selection seed `32039`;
- selected Oracle-like teacher datasets from current T023/T022 workflows at
  budgets 20, 50, and 100 on the same selected source set;
- trainer-input v6 artifacts from the T024 bridge using the budget-100 teacher
  artifact;
- one diagnostic PyTorch checkpoint written under an ignored or external
  artifact path;
- one T027 calibration report comparing that checkpoint with the generated
  trainer artifact.

The PR must report artifact identities and regeneration commands. No large
teacher dataset, trainer JSONL, checkpoint, or report is committed.

## Scope

- If current commands cannot express the required T039 source selection, add or
  extend a focused selected-source manifest/loader so the selection is
  reproducible and reviewable.
- Select source starts only by rule-defined structural metadata from the T039
  contract. The required selected set is all 31 Act 1 Boss starts, all 3 Act 2
  starts, and 64 deterministic Act 1 non-Boss background starts selected with
  seed `32039`.
- Generate current-schema teacher scale-up artifacts at native search budgets
  20, 50, and 100 on exactly the same selected source set.
- Generate teacher-targeted trainer-input records for one selected budget.
- Train one diagnostic checkpoint under the existing `train` optional
  dependency group and a named `narrow_curriculum` override.
- Run teacher-guidance calibration against the generated trainer/checkpoint
  pair.
- Report policy target kind, information regime, source coverage, skipped
  rows, calibration metrics, and broad-training gate status separately.

## Out Of Scope

- Controller promotion or fixed-cohort controller comparison.
- Normal-information claims from Oracle-like teacher data.
- Live-game validation.
- New model architecture work beyond what is required to run the current
  checkpoint contract.
- Treating repeated teacher rows or budget tiers as new source coverage.
- Treating the T031 Act-1-only distribution as broad A20 evidence.
- Treating the T036 Act-1-only smoke distribution as broad A20 evidence.
- Reinterpreting the T039 contract as broad A20 readiness.
- Filtering the selected sources by deck, relic, potion, route, apparent
  winnability, or strategic quality.

## Design Constraints

- Teacher collection remains `full_simulator_state_oracle_like`.
- Required artifact inputs must be reproducible from the T039 contract and
  current commands; do not consume another branch's temporary smoke checkpoint.
- Preserve T039 pool-level and row-level source identity through every teacher,
  trainer, checkpoint, and calibration artifact.
- Preserve selected-source coverage separately from teacher row count, root row
  count, repeated budget tiers, and optimization weight.
- Include all required rare source starts unless deterministic restore or
  schema validation fails closed; exclusions must be rule-defined and reported.
- Mixed policy target kinds, source mismatches, missing trainer provenance, or
  incompatible checkpoints must fail closed.
- PyTorch remains optional under the `train` dependency group, but this task's
  checkpoint step must run in an environment where that group is available.

## Deliverables

- Exact artifact-generation commands and identities in the PR report.
- A selected-source manifest or equivalent report with selected versus
  available counts for Act 1 Boss, Act 2, and Act 1 non-Boss background starts.
- Teacher scale-up, bridge, checkpoint, and calibration summaries.
- Documentation impact notes, usually an `experiment_log.md` entry if no code
  changes are required.
- Focused tests only if code changes are needed.

## Acceptance Criteria

- The selected source set includes all 31 accepted Act 1 Boss starts, all 3
  accepted Act 2 starts, and 64 deterministic Act 1 non-Boss background starts,
  unless the PR fails closed with a rule-defined validation reason. The
  background selection seed, ordering, selected checkpoint ids, and structural
  counts are reported.
- Every generated teacher/trainer/checkpoint/calibration artifact has a
  reported source identity and regeneration command.
- The same selected source identities are used at teacher budgets 20, 50, and
  100.
- Calibration uses a checkpoint whose trainer-input SHA-256 provenance matches
  the generated trainer artifact.
- Teacher target kind, information regime, source coverage, and skipped rows
  are reported separately.
- T009 broad-training readiness is reported only through the configured gate
  and remains distinct from the named `narrow_curriculum` diagnostic override.
- No controller-strength, normal-information, or live-game claim is made.
