# T031: A20 Coverage Refresh And Data Gap Report

## Objective

Refresh A20 battle-start coverage evidence at a broader-than-smoke scale using
the current T021/T008 reporting surfaces, then publish a data-gap report that
decides which strata still block broad training, teacher refreshes, and later
fixed A20 comparisons.

This is the first post-M1 data task. It measures coverage; it must not train a
model, collect a new controller benchmark, or claim A20 strength.

## Current Main Baseline

Current `main` has current-schema A20 coverage reporting, conservative
constructed supplements, structured outcomes, public-context propagation,
fixed restored-battle evaluation, Oracle teacher reporting, and the M1
model-guided Oracle-like sandbox. The accepted coverage and teacher evidence
is still smoke-scale and mostly Act 1, so broad training remains closed.

## Dependencies

- T030 is complete.

## Inputs And Artifacts

Required inputs are generated by documented commands from the current
repository and pinned `sts_lightspeed` source:

- one A20 natural battle-start pool generated with
  `--lightspeed-battle-start-pool`;
- one optional constructed supplement audit generated with
  `--lightspeed-constructed-battle-start-audit`;
- one current-schema coverage report generated with
  `--lightspeed-a20-battle-start-coverage`.

Generated pools, constructed artifacts, and reports must stay under an ignored
artifact directory or an external path. The PR must report artifact paths,
schema ids, SHA-256 identities where emitted or easily computed, source
manifest identity, and regeneration commands. No large generated artifact is
committed.

## Scope

- Run a reproducible A20 coverage refresh with at least 50 source episodes,
  ascension 20, and a reported simulator step cap.
- Include natural battle-start counts, unique natural source counts,
  structural coverage, restore evidence, public-context availability,
  structured-outcome availability, and broad-training gate cells.
- Include constructed supplement counts if the constructed audit succeeds for
  the generated source pool.
- Preserve natural, sampled optimization-weight, and constructed supplement
  distributions separately.
- Add a dated `docs/experiment_log.md` entry summarizing the measurement and
  linking it to the ignored/external artifact identities reported in the PR.

## Out Of Scope

- New command surfaces or schema changes unless an existing bug blocks the
  measurement.
- Broad PyTorch training or checkpoint promotion.
- Oracle teacher collection, trainer-input generation, or checkpoint training.
- Fixed-cohort controller comparison.
- Normal-information belief search or live-game validation.
- Treating constructed or repeated sampled rows as new natural source
  coverage.

## Design Constraints

- Real simulator evidence runs through WSL against the pinned
  `sts_lightspeed` source.
- Under-coverage is a reportable result, not a command failure.
- Artifact validation, restore failures, malformed provenance, or constructed
  source mismatches must fail closed.
- A0 data may not satisfy A20 gate cells.
- The report may use native restore mechanics for verification but must not
  expose hidden state to normal controller or model-input records.

## Deliverables

- Exact local documentation checks and WSL commands in the PR report.
- Ignored or external coverage artifacts with reported identities and
  reproduction commands.
- One dated `docs/experiment_log.md` entry with coverage counts, gate outcome,
  known limitations, and the next recommended task dependency.
- No committed large artifacts.

## Acceptance Criteria

- The coverage refresh uses a newly generated A20 source pool with at least 50
  source episodes.
- The report separates natural unique-source coverage from sampled rows and
  constructed supplements.
- The report states per-ascension/per-act T009 broad-training gate results and
  the exact cells that remain closed.
- Restore, public-context, and structured-outcome availability are reported.
- Any constructed supplement rows are counted separately and do not increase
  unique natural source coverage.
- The PR does not claim broad training readiness unless the configured gate
  actually passes.
- No new model, checkpoint, controller, game file, simulator binary, or large
  generated artifact is committed.
